# 0.2.1 - 2026-06-27

## Added

- Added `tp --check-update` to compare the installed version with the latest GitHub Release.
- Added update instructions to the check-update output.
- Added `tp --whats-new` to print embedded Markdown release notes since the previous installed version.
- Added `tp --whats-new --url` to print the latest release notes URL known to the installed tool.
- Added installed-version tracking in `~/.config/timepro-cli/config.json`, including `version`, `previousVersion`, and `installedAt`.
- Added update-check tracking in config, including the last checked time and latest version seen.
- Added `tp info` with version, tenant, user, and update status.
- Added `tp info --detailed` for install history and config diagnostics.
- Added embedded release notes under `release-notes/<version>.md`.
- Added a release-process skill that requires a matching release note before creating a new release.
- Added `release-notes/latest.md` as the release-body symlink for the newest note.

## Changed

- Bumped the package line to `0.2.x`.
- Changed release versioning so the workflow derives the next patch from the latest GitHub Release for the current major/minor line.
- Made release creation use `release-notes/latest.md` as the GitHub Release body while still validating it points at the computed versioned note.
- Updated generated skills to prefer `tp info --json` over `tp --version` so agents surface update guidance during normal use.
- Kept `tp --version` as pure version-only output; installers use `tp info --no-update-check` for silent installed-version tracking.

## Notes

- Patch-zero local builds, such as `0.2.0`, are treated as development builds and do not overwrite the saved installed release version.
